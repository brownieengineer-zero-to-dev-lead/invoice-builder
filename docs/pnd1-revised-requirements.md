# ข้อกำหนดใหม่: ระบบ Payroll + ภ.ง.ด.1

## ภาพรวม

เดิมที่ implement ไปเรียกว่า "ภ.ง.ด.1" แต่จริงๆ แล้วเป็นคนละส่วนกัน:

| สิ่งที่ built แล้ว | สิ่งที่ควรจะเป็น |
|---|---|
| บันทึกเงินเดือนต่อพนักงานต่อเดือน (Payroll Record) | เอกสาร ภ.ง.ด.1 ที่สรุปพนักงานทุกคนรวมกัน ยื่นสรรพากรรายเดือน |

---

## Concept ที่ถูกต้อง

```
Payroll Records (รายพนักงาน / รายเดือน)
    └── แต่ละ record = พนักงาน 1 คน, เดือน 1 เดือน, เงินได้ + ภาษีที่หัก

ภ.ง.ด.1 (รายเดือน — ยื่นสรรพากร)
    └── ดึง Payroll Records ทั้งหมดของเดือนนั้นมาสรุปรวม
    └── 1 เอกสาร = 1 เดือน = พนักงานทุกคน
```

---

## Feature A: Payroll Records (สิ่งที่ built แล้ว — ปรับชื่อ)

**วัตถุประสงค์**
บันทึกเงินเดือนและภาษีรายพนักงานต่อเดือน ใช้เป็นข้อมูลต้นทางสำหรับ ภ.ง.ด.1 และ 50 ทวิ

**ชื่อ menu:** รายการเงินเดือน (Payroll Records) — ไม่ใช่ ภ.ง.ด.1

### ข้อมูลต่อ Record

| ฟิลด์ | รายละเอียด |
|---|---|
| พนักงาน | required |
| เดือน / ปี | required |
| เงินได้ | default = เงินเดือนตั้งต้น, แก้ได้ |
| ภาษีที่หัก | คำนวณอัตโนมัติ, แก้ได้ |

### Business Rules
- 1 พนักงาน ต่อ 1 เดือน มีได้ 1 record
- ถ้าเดือนนั้นมี record อยู่แล้ว → แจ้งเตือนก่อน edit

---

## Feature B: ภ.ง.ด.1 (Menu ใหม่)

**วัตถุประสงค์**
สรุปเงินเดือนพนักงานทุกคนประจำเดือน เพื่อ export เอกสาร ภ.ง.ด.1 ยื่นกรมสรรพากร

**ใช้งาน:** กด Generate ทุกสิ้นเดือน → ระบบดึง Payroll Records ของเดือนนั้นมารวม → export PDF

### Flow การใช้งาน

```
1. ผู้ใช้เลือกเดือน / ปี / ธุรกิจ
2. ระบบ query Payroll Records ที่ตรงกัน
3. แสดง preview ตารางพนักงานทุกคน + ยอดรวม
4. ผู้ใช้ export PDF เพื่อยื่นสรรพากร
```

### ข้อมูลใน ภ.ง.ด.1

**Header (ข้อมูลบริษัท)**

| ฟิลด์ | ที่มา |
|---|---|
| ชื่อบริษัท / ผู้จ่ายเงินได้ | จาก Business |
| เลขประจำตัวผู้เสียภาษี | จาก Business |
| ที่อยู่ | จาก Business |
| เดือน / ปีที่ยื่น | ผู้ใช้เลือก |

**ตารางรายละเอียดพนักงาน**

| คอลัมน์ | ที่มา |
|---|---|
| ลำดับ | auto |
| ชื่อ-สกุลพนักงาน | จาก Employee master |
| เลขประจำตัวผู้เสียภาษี | จาก Employee master |
| เงินได้ | จาก Payroll Record ของเดือนนั้น |
| ภาษีที่หัก | จาก Payroll Record ของเดือนนั้น |

**Summary (ยอดรวมท้ายเอกสาร)**

| ฟิลด์ | คำนวณ |
|---|---|
| จำนวนพนักงาน | นับ rows |
| รวมเงินได้ทั้งหมด | sum ทุก row |
| รวมภาษีที่หักทั้งหมด | sum ทุก row |

### Business Rules

- ภ.ง.ด.1 ไม่มี record แยก — เป็นแค่ view ที่ generate จาก Payroll Records
- ถ้าพนักงานบางคนยังไม่มี Payroll Record ของเดือนนั้น → แจ้งเตือน แต่ยัง export ได้ (ใส่เฉพาะคนที่มีข้อมูล)
- ถ้าเดือนนั้นไม่มี Payroll Record เลย → แจ้งให้กรอกข้อมูลก่อน

### หน้า ภ.ง.ด.1

- **เลือกเดือน / ปี / ธุรกิจ** (ด้านบน)
- **ตารางพนักงาน** ที่ดึงมาจาก Payroll Records
- **ยอดสรุป** ด้านล่าง
- **ปุ่ม Export PDF**
- (optional) ปุ่ม "กรอกข้อมูลที่ขาด" → ลิงก์ไปยัง Payroll Records page พร้อม pre-filter เดือนนั้น

## Sidebar Structure ที่ปรับใหม่

```
พนักงาน
├── รายการเงินเดือน      ← (สิ่งที่ built แล้ว ปรับชื่อจาก ภ.ง.ด.1)
├── ภ.ง.ด.1              ← (menu ใหม่ — สรุปรายเดือน)
└── 50 ทวิ
```

## Feature C: 50 ทวิ (ปรับ source ข้อมูล)

**วัตถุประสงค์**
ออกหนังสือรับรองการหักภาษี ณ ที่จ่ายให้พนักงานรายปี โดยดึงข้อมูลจาก Payroll Records (ไม่ใช่ ภ.ง.ด.1 โดยตรง)

### Flow การใช้งาน

```
1. ผู้ใช้เลือกพนักงาน / ปีภาษี
2. ระบบ query Payroll Records ทั้งปีของพนักงานคนนั้น
3. แสดง preview รายเดือน (ม.ค.–ธ.ค.) + ยอดรวม
4. ผู้ใช้แก้ไข / เพิ่ม manual ได้
5. Export PDF
```

### ข้อมูลที่ดึงจาก Payroll Records

| ฟิลด์ใน 50 ทวิ | ที่มา |
|---|---|
| รายการเงินได้รายเดือน | Payroll Records ของปีนั้น |
| เงินได้รวม | sum จาก Payroll Records |
| ภาษีที่หักรวม | sum จาก Payroll Records |

### Business Rules
- ถ้า Payroll Records ไม่ครบ 12 เดือน → แจ้งเตือน แต่ยังสร้างได้
- เพิ่มรายการ manual ได้กรณีมีเงินได้พิเศษนอกระบบ

## ความสัมพันธ์ข้อมูล

```
Employee (master)
    └── Payroll Records (รายพนักงาน / รายเดือน)
              ├── ภ.ง.ด.1 View (aggregate ทุกคน ทั้งเดือน → export PDF ยื่นสรรพากร)
              └── 50 ทวิ (aggregate ทีละคน ทั้งปี → export PDF รายคน)
```
